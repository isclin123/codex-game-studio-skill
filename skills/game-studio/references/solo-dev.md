# Solo Dev Workflow

Use for solo developers and very small teams.

## Rhythm

- Keep a single playable build at all times.
- Use weekly goals: one mechanic, one content slice, one polish pass.
- Keep scope aggressively small until the core loop is fun.
- Prefer reusable data and simple tools over large frameworks.

## Priorities

- First: playable loop.
- Second: feedback and readability.
- Third: save/restart/progression.
- Fourth: content and polish.
- Last: complex meta systems.

## Risk Control

- Cut features that require many disciplines unless they define the game.
- Use placeholders with strong readability.
- Keep a devlog or changelog for momentum and playtest history.

# Accessibility

Use for readable UI, remapping, subtitles, colorblind modes, reduced motion, difficulty assists, and input options.

## Baseline

- Text must be readable at target resolution.
- Do not communicate critical information by color alone.
- Provide pause, restart, volume controls, and subtitle/caption support where relevant.
- Avoid rapid flashing; provide reduced motion options for intense effects.

## Input

- Support keyboard remapping for desktop games.
- Support controller prompts when controllers are supported.
- Mobile touch targets should be large and spaced.
- Avoid mandatory multi-finger gestures unless the game is built around them.

## Visual

- Colorblind-safe palettes for enemy/team/state colors.
- Adjustable text scale when UI is text-heavy.
- High contrast mode for dense strategy, card, or puzzle games.

## Gameplay Assists

- Difficulty options, aim assist, timing assist, invulnerability toggles, skip options, or puzzle hints when genre-appropriate.
- Keep assists explicit and respectful; do not shame the player.

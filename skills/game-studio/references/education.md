# Education Workflow

Use for students, teachers, workshops, tutorials, and learning projects.

## Teaching Goals

- Pick one learning objective per project: loop, physics, AI, UI, networking, shaders, etc.
- Keep code readable and explicit.
- Prefer small complete games over large incomplete systems.
- Include reflection questions and extension tasks.

## Student Project Structure

- `README.md` with controls and goal.
- `src/` or engine-native scripts.
- `assets/` with licensed or self-made assets.
- `NOTES.md` explaining what was learned.

## Assessment

- Does it run?
- Is the core loop clear?
- Are controls and feedback understandable?
- Did the student document design decisions and limitations?

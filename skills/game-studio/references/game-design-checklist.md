# Game Design Checklist

Use this when inventing or expanding gameplay.

## One-Minute Loop

Define:
- Player verb: move, aim, jump, match, dodge, build, steer, solve
- Objective: survive, reach goal, clear board, defeat waves, collect targets
- Pressure: timer, enemies, hazards, limited moves, resource drain
- Feedback: sound/visual hit, particles, score popup, screen shake, animation
- Progression: increasing speed, more hazards, new level, combo, upgrades

## Minimum Complete Prototype

A playable prototype should include:
- Start or immediate play state
- Clear controls
- HUD with score/health/time/objective
- Failure condition
- Win or wave/level completion condition
- Restart
- At least one tuned level, arena, or wave pattern

## Feel Pass

Improve:
- Input responsiveness
- Acceleration/friction/jump feel
- Collision forgiveness
- Enemy spawn fairness
- Readability of threats and rewards
- Juicy feedback: particles, color flash, shake, squash, trail

## Accessibility Basics

Prefer:
- High contrast between player, enemies, hazards, and background
- Text labels in addition to color-only state
- Keyboard controls and clickable/touch alternatives when practical
- Motion that does not obscure critical UI

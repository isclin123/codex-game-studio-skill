# Steam Publishing Reference

Use for PC game release planning, Steam store setup, demos, Next Fest, achievements, depots, and launch readiness.

## Steam Readiness

- Define supported OS, minimum specs, input support, language support, and save behavior.
- Prepare capsule art, screenshots, trailer, short description, long description, tags, and mature content settings.
- Decide whether to ship a demo, prologue, early access build, or full release.

## Build And Depot Checklist

- Produce a clean release build separate from dev builds.
- Verify executable launch from a clean folder.
- Include required redistributables.
- Test save path, settings path, and controller behavior.
- Set up depots for Windows/macOS/Linux only when actually supported.

## Steamworks Features

- Achievements: define unlock rules and offline sync expectations.
- Cloud saves: define file paths and conflict behavior.
- Leaderboards: validate anti-cheat or trust boundaries.
- Workshop: include only when user content is a real product feature.
- Remote Play Together: evaluate for local multiplayer games.

## Launch Plan

- Store page live before launch for wishlists.
- Demo or festival plan if discovery matters.
- Press kit: logo, screenshots, trailer, fact sheet, contact, keys policy.
- Community: forums, Discord, bug reports, patch cadence.

## Final Answer Requirements

When finishing a Steam publishing task, include:

- build output
- store assets still needed
- Steamworks features touched
- test matrix
- release risks

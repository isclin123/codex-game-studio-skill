# Playtesting

Use for playtest plans, feedback forms, observation notes, retention metrics, and tuning loops.

## Test Types

- Smoke test: does the game launch and can the player complete the core loop?
- Usability test: does the player understand controls, goals, feedback, and UI?
- Balance test: is difficulty, pacing, and reward timing right?
- Content test: are levels, enemies, and progression working?
- Release test: install, update, save migration, platform compliance.

## Session Plan

- Define what question the test answers.
- Give only the context a real player would have.
- Watch where players hesitate, not just what they say.
- Record device, build, session length, failure points, and quotes.

## Feedback Questions

- What did you think your goal was?
- What felt confusing?
- What felt satisfying?
- Where did you want to stop?
- What would you try next?

## Metrics

- Time to first input.
- Time to first success/failure.
- Restart count.
- Level completion rate.
- Session length and return intent.

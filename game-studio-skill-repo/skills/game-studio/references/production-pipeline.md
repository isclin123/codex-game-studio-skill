# Production Pipeline

Use for game projects larger than a quick bug fix.

## Milestone Ladder

1. Concept - one paragraph pitch, pillars, reference games, target platform.
2. Paper prototype - rules, loop, win/loss, economy, enemy/player verbs.
3. Graybox - ugly but playable level or arena.
4. Vertical slice - one polished chunk showing final intent.
5. Content expansion - levels, enemies, progression, UI flows.
6. Beta - feature complete, tuning, bug fixing.
7. Release candidate - platform checks, save compatibility, performance, packaging.

## Vertical Slice Requirements

- One complete player objective.
- Representative controls and camera.
- Representative art direction or at least strong placeholder readability.
- Final-ish HUD and feedback language.
- One difficulty curve moment.
- Win/loss/restart.
- Measurable performance target.

## Task Breakdown Template

- Design: loop, rules, balance table, levels.
- Engineering: architecture, gameplay systems, UI, save, tools.
- Art: style guide, asset list, implementation format.
- Audio: SFX list, music direction, mixer groups.
- QA: smoke tests, playtest script, device/platform matrix.
- Production: milestones, risks, dependencies.

## Definition Of Done

A game task is not done until:
- It is playable in the intended scene or route.
- It has clear feedback.
- It handles restart/reload.
- It does not break existing controls.
- It has at least one verification step.
- The final answer tells the user how to run it.
